## Add Participant (Modal or Page)

### Title
**Add Participant**

### Subtitle
Create a new participant account and set a temporary password. They’ll be required to set a new password at first login.

### Field labels + helper text
**Full name** (required)  
Helper: “First and last name.”

**Date of birth** (required)  
Helper: “Used for identity confirmation and reporting.”

**Email** (required)  
Helper: “This will be their login username.”

**Phone number** (optional/required)  
Helper: “Used for account recovery and optional reminders.”

**Assign coach** (optional)  
Helper: “You can assign a coach now or later.”

**Temporary password** (required)  
Helper: “Participant will be forced to reset this password at first login.”

**Confirm temporary password** (required)

Checkbox (default ON):
✅ **Force password reset at first login**  
Helper: “Recommended for security.”

### Password requirements text (keep simple)
“Use at least 10 characters. Avoid common words.”

### Buttons
Primary: **Create Participant**  
Secondary: Cancel

### Inline validation messages
- Email already exists:  
  “That email is already in use. Try another email or search for the participant.”
- Weak password:  
  “Password is too weak. Use at least 10 characters.”

## Small UX Rules (important)
- Always default the “Force password reset” checkbox to ON.
- Never show or store plaintext passwords in logs.
- If you display the temporary password once, do it only on the success screen.
- Add a “copy” button wherever credentials appear.
- Provide clear error messages (unique email, invalid DOB, weak password).
- Confirm destructive actions (Deactivate).
